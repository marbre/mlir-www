79 /// Populate patterns for SCF software pipelining transformation.
80 /// This transformation generates the pipelined loop and doesn't do any
81 /// assumptions on the schedule dictated by the option structure.
82 /// Software pipelining is usually done in two part. The first part of
83 /// pipelining is to schedule the loop and assign a stage and cycle to each
84 /// operations. This is highly dependent on the target and is implemented as an
85 /// heuristic based on operation latencies, and other hardware characteristics.
86 /// The second part is to take the schedule and generate the pipelined loop as
87 /// well as the prologue and epilogue. It is independent of the target.
88 /// This pattern only implement the second part.
89 /// For example if we break a loop into 3 stages named S0, S1, S2 we would
90 /// generate the following code with the number in parenthesis the iteration
91 /// index:
92 /// S0(0) // Prologue
93 /// S0(1) S1(0) // Prologue
94 /// scf.for %I = %C0 to %N - 2 {
95 /// S0(I+2) S1(I+1) S2(I) // Pipelined kernel
96 /// }
97 /// S1(N) S2(N-1) // Epilogue
98 /// S2(N) // Epilogue
99 void populateSCFLoopPipeliningPatterns(RewritePatternSet &patterns,
100  const PipeliningOption &options);
